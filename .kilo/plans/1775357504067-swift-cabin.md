## Considerações Técnicas

### Dependências
- Utilizar `jsr:@std/zip@^0.224.0` para ZIP (nativo Deno, sem dependências externas)
- Para 7z utilizar o binário do sistema através de `Deno.Command()` pois não existe implementação nativa em Deno viável para arquivos grandes

### Tratamento de erros
- Todos os erros devem retornar mensagens claras para o usuário
- Arquivos parciais devem ser limpos em caso de falha
- Hash é verificado ANTES da descompactação, então não precisamos re-verificar

### Performance
- Descompactação ocorre diretamente no diretório do store
- Arquivo compactado é deletado imediatamente após extração bem sucedida para economizar espaço
- Para arquivos grandes o processo deve ser streaming para não carregar tudo em memória
